# pipy documentation

Pipy is a local-first coding-agent harness experiment. The product direction is
`pipy-native`: a Python runtime that owns provider access, tool boundaries,
session semantics, and privacy-conscious archive metadata.

Start with these project documents:

- [Backlog](backlog.md): current product planning, completed slices, near-term
  priorities, and deferred boundaries.
- [Harness Spec](harness-spec.md): architecture, native runtime direction,
  adapter boundaries, and design rationale.
- [Session Storage](session-storage.md): session archive layout, metadata-only
  capture rules, sync behavior, and privacy policy.
