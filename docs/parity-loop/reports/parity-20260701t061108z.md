## What Changed

This slice completed the `openai-completions` `string-thinking` reasoning request shape. Models that explicitly declare `compat.thinkingFormat = "string-thinking"` now send a top-level string `thinking` field instead of OpenAI-style `reasoning_effort`.

User-visible request behavior now matches Pi for this format:

- active reasoning sends `thinking: <mapped level>` and falls back to the requested level when no map entry exists;
- off or unset reasoning sends `thinking: "none"` by default;
- an explicit `thinkingLevelMap.off` string overrides that default;
- an explicit `thinkingLevelMap.off = null` suppresses the off-state field entirely;
- `supportsReasoningEffort` is ignored for this format, so `reasoning_effort` is never emitted.
