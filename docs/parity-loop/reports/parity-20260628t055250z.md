## What Changed

This slice shipped user-facing settings and keybindings documentation so a pipy user can now configure the native runtime without reading maintainer specs or Pi source docs.

The new settings guidance explains where pipy reads global and project `settings.json` files, how project settings deep-merge over global settings, when to use `/settings` versus `/reload`, and which common Pi-shaped settings are active, accepted for future compatibility, or intentionally pipy-specific. It also makes the privacy boundary explicit: provider credentials belong in auth stores or environment variables, not in `settings.json`.

The new keybindings guidance explains the global `keybindings.json` location, key syntax, reload/hotkey discovery workflow, migration from older flat action names, and the shipped namespaced action ids/default shortcuts for editor, input, application, session, model, tree, and scoped-model-selector actions. The docs index, site navigation, changelog, backlog, and parity audit now treat settings/keybindings docs as shipped user documentation rather than an open parity gap.
