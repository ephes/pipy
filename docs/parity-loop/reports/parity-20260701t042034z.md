## What Changed

Extensions can now use `send_message` / `sendMessage` with `deliverAs: "steer"` or `deliverAs: "followUp"` to place custom-message content onto pipy's provider-visible steering/follow-up queue. The message is still recorded as a bounded custom message for local display first, but its `content` is also drained as model input in the same path as interactive steering and follow-up prompts. Steering content drains before follow-up content, and queued text that begins with `/` is treated as provider input rather than a local slash command.

This completes the previously deferred streaming delivery modes for extension custom messages alongside the already-shipped `triggerTurn` and `deliverAs: "nextTurn"` behavior.
