## What Changed

This slice added Pi-shaped custom editor component state to extension UI contexts. Live command and shortcut extensions can now call `ctx.ui.set_editor_component(factory)` / `setEditorComponent(factory)` and later retrieve the same opaque factory with `ctx.ui.get_editor_component()` / `getEditorComponent()`. Passing `None` clears the live store.

Headless extension contexts intentionally match Pi RPC behavior: setting a component is a deterministic no-op and getting one returns `None`. The stored live value is in-memory only; pipy does not call the factory, render it, serialize it, or expose it through safe metadata.
