## What Changed

Extensions that call `ctx.ui.paste_to_editor(text)` now get Pi-style live editor paste semantics in the native TUI. Instead of replacing the entire prompt draft, pasted text is inserted at the current cursor position, existing text before and after the cursor is preserved, pasted newlines remain literal, and the cursor advances to the end of the inserted text.

`ctx.ui.set_editor_text(text)` remains the replacement operation. This makes the two editor helpers semantically distinct: extensions can either overwrite the draft intentionally or paste into the user's current draft as Pi does.

<details>
<summary>Why does this matter for Pi parity?</summary>

Pi routes extension paste through the editor's bracketed-paste path, so pasted content joins the user's current draft instead of discarding it. Pipy's live TUI now matches that user-visible behavior.
